## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/prism.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Install dependencies: `npm install`
5. Make your changes
6. Run tests: `npm test`
7. Run type check: `npm run type-check`
8. Run linter: `npm run lint`
9. Commit your changes: `git commit -m "feat: your feature description"`
10. Push to your fork: `git push origin feature/your-feature-name`
11. Open a Pull Request

## Code Style

### TypeScript
- Use strict TypeScript with no `any` types
- Follow ESLint rules
- Add JSDoc comments for public APIs
- Use descriptive variable names

### Rust
- Follow Clippy lints
- Use inline format arguments
- Add comprehensive tests
- Document public APIs

### Commit Messages

Follow Conventional Commits:
- `feat:` - New feature
- `fix:` - Bug fix
- `docs:` - Documentation only
- `style:` - Code style changes
- `refactor:` - Code refactoring
- `test:` - Adding tests
- `chore:` - Maintenance tasks

## Testing

- Write unit tests for new features
- Ensure all tests pass before submitting
- Aim for 80%+ code coverage
